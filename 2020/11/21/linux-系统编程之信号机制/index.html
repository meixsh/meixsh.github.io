linux-系统编程之信号机制

发表于 2020-11-21 分类于 linux系统编程 阅读次数：
本文字数： 568 阅读时长 ≈ 1 分钟

信号的概念和机制

  • 信号共性
    • 简单
    • 不能携带大量消息
    • 满足条件才发送
  • 信号的特质
    • 信号是软件层面上的中断。一旦信号产生后，无论程序执行到什么位置，必须立即停止运行，处理信号，处理结束，再继续执行后续指令；
    • 所有信号的产生及处理全部都是由内核完成的。
  • 信号的三种状态
    • 产生信号
      1. 按键产生
      2. 系统调用产生
      3. 软件条件产生
      4. 硬件异常产生
      5. 命令产生
    • 未决状态
      • 产生与递达之间的状态
    • 递达状态
      • 产生并且递达到进程。直接被内核处理掉。
  • 信号处理方式
    • 执行默认处理动作
    • 忽略
    • 捕捉（自定义）
  • 阻塞信号集（信号屏蔽字）
    • 本质：位图
    • 用来记录信号的屏蔽状态。一旦信号被设置为阻塞，在解除阻塞前，一直处于未决状态
  • 未决信号集
    • 本质：位图
    • 用来记录信号的处理状态。该信号集中的信号，表示信号已经产生并递达，但尚未被处理
  • kill -l 查看当前系统中的常规信号
  • 信号四要素
    • 编号
    • 名称
    • 对应事件
    • 默认处理动作

产生信号的函数及命令

  • kill系统调用函数
    • int kill(pid_t pid, int signum)
    • 参数
      • pid
        • >0 发送信号给指定进程
        • =0 发送给跟调用kill函数的那个进程处于同一进程组的进程
        • <-1 取绝对值，发送给该绝对值所对应的进程组的所有组员
        • =-1 发送信号给有权限发送的所有进程
      • signum
        • 待发送的信号
    • 返回值
      • 成功 0
      • 失败 -1 errno
  • kill命令
    • kill -9 pid 杀死pid进程
    • kill -9 -groupname 杀一个进程组

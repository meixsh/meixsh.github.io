CPU位数

  • x86 32bit Linux内核
  • CPU的位数是32位的
    • CPU是用来取指执行的一个单元，内部包含ALU部件用来计算，一般说来，CPU的位数就是指ALU数据单元的运算宽度，也即数据总线条数。x86 32位架构下，地址总线与数据总线都是32位。但是在16位以及8位早期8086等架构下，数据总线与地址总线条 数则不相同
      • 16位，数据总线16位，地址总线20位
      •  8位，数据总线 8位，地址总线16位
  • IBM关于物理、虚拟以及透明的透彻解释
    • If you can see it and it’s there, then it’s real. If you can see it and it’s not there, then it’s virtual. If you can’t see it and it’s there, then it’s transparent. If you can’t see it and it’s not there, then you erased it.
    • 如果你可以看见一样东西，并且它就在那边，那么它就是真的。如果你可以看见一样东西，并且它实际上不存在，那么它就是虚拟的。如果你看不见一样东西，并且它就在那边，那么它就是透明的。如果你看不见一样东西，并且它不在那边，那么是因为你把它删除了。

虚拟地址空间

  • 基于x86 32位架构
  • 虚拟地址空间大小 232 = 4G

  • gcc -c main.c -o main.o

  • objdump -h main.o

  • readelf -h main.o

    • 查看elf header信息
    • size of this header 52字节 0x34

  • file main.o

    • relocatable

  • readelf -S main.o

    • 查看elf文件的段表信息

  • objdump -d main.o

    • 打印反汇编结果

  • objdump -s main.o

    • 打印段的内容

  • objdump -t main.o

    • 打印符号表

  • 链接阶段

    • 所有obj文件的global符号进行处理
    • local的符号不作任何处理
    • 合并所有obj文件的段，并调整段偏移和段长度，合并符号表，进行符号解析
      • 符号解析：所有obj符号表中对符号引用的地方都要找到该符号定义的地方
    • 链接的核心
      • 符号的重定位

  • 程序的运行 进程 ./a.out

    • 创建虚拟地址空间到物理内存的映射（创建内核地址映射结构体），创建页目录和页表
    • 加载代码段和数据段
    • 把可执行文件的入口地址写到CPU的PC寄存器里面

  • readelf -l a.out

    • 查看program headers

  • strace ./a.out

    • 追踪可执行文件加载映射的过程

  • cat /proc/pid/maps

    • 查看某个进程对应的虚拟地址空间信息

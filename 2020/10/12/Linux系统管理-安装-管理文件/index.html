Linux系统管理-安装 & 管理文件

发表于 2020-10-12 更新于 2020-10-13 分类于 linux系统管理 阅读次数：
本文字数： 7.2k 阅读时长 ≈ 7 分钟

安装Linux系统

选择安装镜像

  • 32位机 i386, 64位机 x86_64
  • 64bit Checker 工具确定windows主机是否是64位机
  • 镜像地址 选择对应的镜像文件以及MD5校验码文件
    • MD5校验工具 确认文件完整性

VMware安装虚拟机

  • 指定磁盘容量时，选择单个文件存储虚拟硬盘

  • 不要勾选UTC时间，会造成时间显示与本地显示不一致的问题

  • 自定义硬盘分区

    • 分区表：一个硬盘仅拥有64字节的分区表，用来记录硬盘分区信息，但是每个分区信息大小是16字节，因此每个硬盘最多可以分成4个主分区
    • 4个主分区中可以有一个被替换为扩展分区，扩展分区内部可以继续划分逻辑分区，其数量理论上无限制
    • 机器第1块硬盘被称为sda，第2块硬盘被称为sdb… 其中，sd(SCSI Device)
      • sda进行分区之后，主分区依次命名为sda1、sda2、sda3、sda4；逻辑分区起始号码总是为sda5，与前面主分区数量无关
    • 格式化分区，规定数组组织方式，创建文件系统
      • windows：fat32，ntfs
      • centos: ext4, swap
    • swap分区：与虚拟内存有关，使用SWAP文件系统
      • swap分区不能挂载到目录树上，所以目录树上看不见swap分区
      • redhat公司关于swap分区内存大小划分建议，根据实际内存大小划分
    • 创建两个分区，根分区以及swap分区
      • 创建swap分区，选择标准分区，文件系统类型选择swap，虚拟机内存分配4g，swap分区大小设置为4096MB
      • 创建根分区，选择标准分区，挂载点选择/, 文件系统选择ext4，选择使用全部可用空间，其他默认

  • grub引导转载程序安装，默认安装

    • grub负责从硬盘分区找到内核，然后将内核装载进内存，grub可以同时帮助引导多种操作系统，windows以及BSD系列unix系统，多系统安装时有用，需要配置

  • 系统软件包安装选择

    • 选择basic server选项，然后点击现在定制按钮
    • 初学者左栏建议勾选桌面，右边勾选除KDE桌面之外的所有选项
    • 根据需要勾选web服务器以及数据库等

  • 进入系统后，开始一些系统基本配置，建议不选择kdump，作为一个使用者而言而非核心开发人员，会浪费空间

shell

bash shell (The GNU’s Bourne Again Shell)

命令

  • 一个二进制可执行文件
  • 一个shell脚本文件
  • 一个shell内建函数

命令选项

  • 控制命令行为的开关
  • 选项一般前面有一个 ‘-‘ 或者 ‘–’符号加上一个或多个字母的代码

命令参数

  • 参数一般用来向命令传递一些运行时所需的信息，一般来说是一个字符串

三个简单命令

  • whoami

    • 显示当前登录用户的用户名
    • 最简单的命令，不接受参数，只有两个基本选项
    • –help选项
    • –vesion选项

  • echo

    • echo "hello world" 屏幕回显
    • echo -n "hello world" 回显不换行
    • echo -e "hello world\n\n" 输出包括反斜杠定义的转义字符

  • nano

    • ctrl + o 填入文件名回车保存
    • ctrl + g 帮助文档
    • ctrl + x 退出帮助文档

  • 命令历史

    • 键盘上下箭头

  • 命令补全

    • tab

帮助文档man

  • man who
  • COMMANDS 命令中的内部指令
  • FILES 与命令相关的文件
  • 上下箭头移动一行，page up/down 上下翻页
  • /字符串 光标处向下查找
    • n 下一个匹配处
    • N 上一个匹配处
  • q 退出
  • man文档的9大类
    • 1 可运行的指令或可执行文件的帮助文档
    • 2 系统核心可使用的函数与工具等的帮助文档
    • 3 一些常用函数与库的帮助文档
    • 4 设备文件的说明
    • 5 配置文件或者是某些文件的格式说明
    • 6 游戏程序的帮助文档
    • 7 惯例与协议，如文件系统、网络协议等的说明
    • 8 系统管理员可用的管理命令的帮助文档
    • 9 跟系统核心有关的文件的帮助文档
  • man 1 read 和 man 2 read, 不加数字默认从较小数字中寻找相关内容

关闭和重启linux

正确步骤

  • 查看在线用户和在线服务
  • 通知用户关机/重启原因和时间安排
  • 下达关机或重启命令
    • 需要root权限
    • su - root
    • shutdown -h 10 # 10min后关机
    • shutdown -h HH:MM
    • shutdown -h now # 立即关机
    • shutdown -h 时间 字符串字符串参数可以解释关机原因
    • -r 选项重启
    • shutdown -c 取消关机或重启
  • 不安全的关机重启命令
    • reboot = shutdown -r now
    • halt = shutdown -h now
    • poweroff = halt + 切断电源指令 （不支持ACPI的机器此指令相当于halt）
    • 这三个命令在关闭系统时，删除所有进程，而非平稳终止这些进程，此时若还有其他用户在工作时，会造成数据丢失，不建议使用

管理文件

标准文件系统架构（Filesystem Hierarchy Standard/FHS）

  • 规定linux倒树形根目录下有哪些目录，这些目录存放什么数据
  • 根目录下至少有如下十二个目录
    /bin /boot /dev /etc /home /lib /root /sbin /srv /tmp /opt /media
    • /sbin 放置root用户才能执行的命令文件
    • /srv 服务启动之后需要访问的数据，如web服务启动之后所需要访问的网页数据默认放置在/srv/www目录下
    • /tmp 临时存放文件目录，任何人都可以访问，重要数据不可放置在此目录下
    • /opt 第三方软件建议安装目录，非本发行版本所提供的软件
    • /media 移动设备相关文件，如光驱、u盘

  • 工作目录
    • pwd命令

切换创建和删除目录

  • 特殊目录符号

    • . 当前工作目录
    • .. 工作目录的上层目录
    • - 前一个工作目录
    • ~ 用户家目录
      • 如有两个用户a和b，当前位于a的家目录，若要切换到b的家目录，可以 cd ~b

  • cd命令

  • mkdir

    • -p 选项创建嵌套目录
      mkdir -p /tmp/dirlevel01/dirlevel02

  • rmdir

    • 只能删除空目录

列出文件和文件属性

  • 列出文件

    • ls [文件名|目录名|NONE]
    • -a 列出全部文件，包括隐藏文件
    • -l 以长格式列出文件，不仅列出文件名，还要列出文件的详细属性信息
      包括七个字段
      第二个字段文件硬链接数目
      第五个字段文件大小单位字节

    -h选项可以人性化显示文件大小，当文件很大的时候有用
    第1,3,4字段一起决定了文件访问权限，第三个字段文件拥有者，第四字段文件从属用户组
    第一个字段十个字符的第一个字符表示文件类型

      + `-` 普通文件
  + `d` 目录文件
  + `c` 字符设备文件
  + `b` 块设备文件
  + `l` 符号链接文件

  • chmod 修改文件访问属性

    • chmod 700 hello.sh

  • chown 修改文件拥有者和从属用户组

    • chown nbcc:stu hello.sh 修改hello.sh的文件拥有者为nbcc，从属组为stu
    • chown stu hello.sh 单单修改拥有者
    • chown :nbcc hello.sh 单单修改从属组

拷贝、剪切、删除和创建文件

  • cp

    • cp [要拷贝的文件|文件列表] [目的文件名|目的目录]
    • 拷贝单个文件
    • 拷贝单个文件并重命名
    • 拷贝多个文件（第一个参数填多个文件名并用空格隔开即可）
    • 拷贝目录 cp -r /tmp/dir01 ~
    • 注意：
      • 目标目录存在同名文件时，cp命令执行后会询问是否覆盖
      • 若有批量同名文件存在，则一次次确认很麻烦，使用下面的命令强制覆盖
        \cp -f /tmp/file01 /tmp/file02 ~
      • cp文件拷贝时文件属性包括文件拥有者以及文件从属组和访问权限位都有可能会发生变化，若要原原本本的拷贝，则使用-p选项, 不过需要登录用户对这个文件有写权限才可以
        cp -p /tmp/sample.sh ~nbcc

  • mv

    • mv [要移动的文件|文件列表|目录] [目的文件名|目的目录]
    • 移动目录时，不同于cp命令，他不需要加-r选项
    • 注意：
      • 同名问题，类似cp命令，建议-f选项，不过命令前面不用\
      • 可以重命名文件

  • rm

    • rm [要删除的文件|文件列表|目录]
    • 删除目录时 rm -R ./dir01
    • -f 不询问，直接删除

  • touch

    • touch [要创建的文件名]
    • 注意：
      • 文件已存在会修改文件的时间戳属性（最后修改时间）

文件链接

  • 符号链接

    • 类似windows快捷方式
    • ln -s [链接指向的文件] [链接名] # symbolic
    • 数据块存放源文件的绝对路径信息，删除源文件，链接文件无法打开

  • 硬链接

    • 在linux中，当划分磁盘分区并格式化的时候，整个分区会被划分为两个部分，即 Inode区 和 Data Block区。linux中随后创建的所有文件都分成两个部分存放，一部分是文件的唯一标识，位于Inode区的一个inode中，包含Inode-number以及属性信息，inode-number与inode是一一对应的。Data Block区中存放文件实际数据。linux中需要通过文件的inode来找到存放文件数据的data block。不可能有两个文件共用同一个inode，自然不会存在有两个文件的inode-number是一样的。将硬盘分区比作一本书，Inode区就是书的目录，data block区就是其内容，通过目录检索内容。

    ls -il -i选项帮助列出文件inode-number，此时第一字段就是inode-number
    简单来说，linux中的倒树形目录就是一张表，有多少文件，就有多少表项，每个表项有两个内容，文件名和inode-number。因此linux中存取文件的过程就是通过目录中文件名找到对应inode-number，进而找到分区中文件的inode接点，最后通过inode找到文件data block区中存放的数据。
    有了这些知识，就可以解释硬链接了，硬链接就是目录中存在两个表项的文件名虽然不一样，但是其inode-number都是一样的，即这两个文件指向同一个分区中的inode，自然也对应分区中同一个数据块，文件inode和数据还是同一份，没有变，只不过文件多了一个名字，这个名字就被称为硬链接。

    • ln [链接指向的文件] [链接名]

    • 注意：
      由于文件系统的限制，硬链接不如符号链接灵活
      强烈不推荐为目录创建硬链接，容易造成目录遍历死循环
      不能跨硬盘分区创建硬链接，不同分区中文件的inode-number不再是唯一的

归档压缩文件

tar命令

  • 古老的命令，tape archive缩写，最初是指将文件备份到磁带上
  • 多个文件前后连在一起构成一个大文件，不进行亚索
  • tar -cf [打包文件名] [要打包的文件/列表] -c 表示创建打包文件，-f 指定打包文件名，因为要打包的文件可能只有一个，所以打包文件名参数必须紧跟选项-f，强烈推荐打包文件名后缀为.tar，最终生成的打包文件会比所有被打包文件之后还要大，这是因为打包时tar命名会添加一些额外信息，再次证明tar命令不进行压缩。
  • 可以打包目录
  • tar -tf ./samplefile.tar 列出打包文件中的所有文件名
  • tar -f ./samplefile.tar --delete tmp/file05 使用delete选项删除打包文件中的指定文件
  • tar -f ./samplefile01.tar -A ./samplefile.tar 合并两个打包文件
  • tar -f ./sampledir01.tar -r ~/fileA.txt 向打包文件中添加新文件
  • tar -xf ./sampledir01.tar -C ./sampledir 解包，默认解包目录时当前目录，可以使用-C选项指定目录

gzip解压缩文件

  • gzip [要压缩的文件|列表]
  • gzip ./sampledir01.tar
  • gzip -d ./sampledir01.tar.gz -d选项解压
  • gzip还可以通过1-9的压缩比数字调节压缩时的压缩比，1是压缩后的文件最大，压缩比(压缩后文件大小/原文件)也是最大的，但是压缩过程最快。默认压缩比为6
    gzip -1 samplefile01

gzip只能单个地压缩文件，而不能将多个文件或者整个目录压缩到一个文件中，所以gzip需要结合tar命令一起使用，为了方便使用，tar命令提供选项方便调用gzip命令

  • 打包压缩
    tar -czf sample.tar.gz /tmp/samplefile01 /tmp/samplefile02
  • 解压缩解包
    tar -xzf sample.tar.gz

查找文件

locate查找较快，但功能较弱。
find功能强大，但查找稍慢。

locate命令

  • locate [文件名的关键字]
  • 查找系统中.tar文件
    locate .tar
    执行后发现查找非常快，你会怀疑是否真的去硬盘找了，实际上locate命令还真的没有去硬盘找，这是因为linux系统会将系统内的所有文件名都记录在一个名为/var/lib/mlocate数据库中，所以查找时locate会通过数据库来查，而不是遍历硬盘查。这样导致locate眼中的世界和真实世界有时候存在偏差，有时会查找到已经删除的数据，有时刚建立的文件无法查找到。这是因为数据库未实时更新造成的。这个数据库默认每天更新一次
  • 通过命令手动更新mlocate数据库
    updatedb

find命令

  • find [查找范围] [查找条件] [动作]
    查找范围缺省当前工作目录

  • 根据文件名查找，未使用通配符则指代精确查找
    find / -name passwd

  • 根据文件类型查找, d目录，f普通文件，c字符设备，b块设备，l符号链接
    find /bin -type l

  • 根据文件时间属性查找，选项共分为三组，共六个

    date
    find /tmp -mtime -3 -ls
    查找/tmp目录下三天内文件内容发生改变的文件
    find /tmp -mtime +7 -ls
    查找/tmp目录下七天之前前文件内容发生改变的文件
    find /tmp -mtime 4 -ls
    查找/tmp目录下4天前的那天文件内容发生改变的文件
    -mmin 选项原理类似，只不过时间单位由天改为了分钟

  • 根据文件大小进行查找

  • 根据文件拥有者、所属组进行查找

  • 根据文件权限进行查找
    find /tmp -perm 754 -ls

  • 联合查找
    上面的示例都是单个条件查找，也可以联合这些条件进行查找
    与 -a 或 -o 非 !
    find /tmp \( -size +1k -a -size -10M -a -mmin -30 -a -type f \) -ls
    这些查找条件需要被圆括号括起来，且括号左右都有空格，且需要转义

  • -exec

  • -OK
    可以使用-OK替换-exec，两者用处基本相同，但是前者以一种更为安全的模式来执行动作命令，再执行每一个命令之前都会给出提示，让用户来决定是否执行，删除文件时可以考虑使用它

通配符

删除/tmp目录下所有后缀为.txt的文件
rm -rf *.txt

  • * * 代表任意长度字符串
  • ？代表任意单个字符
  • []
    • [c1-c2] 匹配c1-c2字符序列中的任意单一字符，一般来说是数字或英文字符序列
      cp /tmp/file[0-9].txt ~
    • [c1,c2,…,cN] 匹配中括号之间字符列表中的任意单一字符
      chmod 700 /tmp/script[a,1,x].sh
  • {}
    • {string1,string2,…,stringN} 匹配string1,string2直至stringN中的某个字符串
      rm -rf {jiaoben,script}*.sh
  • !
    • 一般与[]或{}连用表示取反
      tar -cf file.tar /tmp/file[!0-9].txt

参考链接

查看文本文件

linux中配置文件以及日志文件通常都采用纯文本格式存储，所以高效查看这些文本文件很有学习价值。

cat （concatenate）

  • 连接几个文件
  • 从键盘创建文件
  • 查看文件
    • cat /etc/passwd
    • 一次输出文件全部内容到命令行，如果文件内容较多，超过命令行输出缓冲区，则看不到文件前面内容，其也不提供翻页和查找功能，只适合查看较小的文本文件

less/more

  • less /var/log/messages
  • 上下箭头翻行，page up/down上下翻页
  • /关键字 查找文本
    • n 跳转至下一处
    • N 跳转至上一处
  • q 退出回到命令行

head/tail

  • head -n 20 /var/log/messages
    查看文件messages文件头20行，不足20行全部显示
    缺省-n参数，默认头10行
  • tail -f notes.log
    此命令显示 notes.log 文件的最后 10 行。当将某些行添加至 notes.log 文件时，tail 命令会继续显示这些行。 显示一直继续，直到您按下（Ctrl-C）组合键停止显示
  • 经常用来查看日志文件，因为日志文件都是在头尾不断输出的

hexo + github 搭建个人博客

发表于 2020-10-10 分类于 前端 阅读次数：
本文字数： 6.6k 阅读时长 ≈ 6 分钟

序

说实话，我几年前就想搭建一个属于自己的博客，但是那时候比较懒，没有动手干起来，总觉得会是一件比较复杂的事情，有点害怕自己干得不够好，说白了就是不自信加惰性。就在几年前一次偶然的机会，从同学那里了解到了github可以帮助建立个人博客，但是那时候没有深入研究下，遂一直搁置。最近我打算认真记点学习笔记，但是之前使用的有道云笔记和印象笔记都被我弄得很杂乱，而且他们对于md的支持不够好，所以我下定决心一定要搭建一个属于自己的博客，用来保存及展示自己的学习笔记，一方面精心写一篇博客特别锤炼人的耐心有助于养成好习惯，另一方也会给自己带来成就感。我相信通过日积月累自己一定会有所收获。实际上，我从调研到动手搭建好自己的初代博客，也不过花了一上午的时间，所以，一个看似比较复杂的问题，一旦开始动手干起来了，碰到问题时，只要耐心搜索思考，问题都可以解决的。一定要动手实践，做起来！

简介

github提供博客公开访问的服务器，个人用户在自己电脑上编辑博客文件，再通过hexo框架提供的命令完成博客的测试和发布，最终上传至github上实现最终的博客部署。

如上文，这套逻辑很简单，但是需要依赖几个工具，而且还会有一些细节需要注意。不过hexo真的很强大，通过几个工具打造的环境，用户就可以只专注于博客内容的编写而不用费心部署的复杂过程，只需要熟练掌握几个命令即可，真可谓是极大地减少了用户的负担呐。

依赖工具集及安装步骤

  • github：GitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。GitHub于2008年4月10日正式上线，除了Git代码仓库托管及基本的 Web管理界面以外，还提供了订阅、讨论组、文本渲染、在线文件编辑器、协作图谱（报表）、代码片段分享（Gist）等功能。目前，其注册用户已经超过350万，托管版本数量也是非常之多，其中不乏知名开源项目 Ruby on Rails、jQuery、python 等。
    2018年6月4日，微软宣布，通过75亿美元的股票交易收购代码托管平台GitHub。
    需要注册一个账号。然后创建博客仓库，仓库名有要求,要创建一个和你用户名相同的仓库，后面加github.io，只有这样，将来要部署到GitHub page的时候，才会被识别，也就是http://xxxx.github.io，其中xxx就是你注册GitHub的用户名。勾选创建README。

  • git: 开源分布式版本控制系统。推荐廖雪峰老师Git教程。
    Git下载地址。我是windows系统
    64位机，所以选择64位安装包，基本上就是默认安装。安装完成后，需要将git与自己的github账号绑定，桌面鼠标右键打开Git Bash Here，设置user.name和user。email配置信息
    git config --global user.name "你的GitHub用户名"
    git config --global user.email "你的GitHub注册邮箱"
    生成ssh秘钥文件，命令执行之后，直接三个回车即可，默认不需要设置密码
    ssh-keygen -t rsa -C "你的GitHub注册邮箱"
    使用cat命令查看生成的id_rsa.pub秘钥，并复制秘钥内容
    cat ~/.ssh/id_rsa.pub
    点击github右上角自己的小人图像，依次点击settings→SSH and GPG keys，新建new SSH Key。Titile随意填，将刚才复制的id_rsa.pub内容粘贴到下面的Key方框中，最后点击Add SSH key。检测公钥是否设置成功
    ssh git@github.com
    之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

  • node.js: 是能够在服务器端运行JavaScript的开放源代码、跨平台JavaScript运行环境。Node.js采用Google开发的V8运行代码，使用事件驱动、非阻塞和异步输入输出模型等技术来提高性能，可优化应用程序的传输量和规模。这些技术通常用于资料密集的即时应用程序。Node.js大部分基本模块都用JavaScript语言编写。在Node.js出现之前，JavaScript通常作为客户端程序设计语言使用，以JavaScript写出的程序常在用户的浏览器上运行。Node.js的出现使JavaScript也能用于服务端编程。Node.js含有一系列内置模块，使得程序可以脱离Apache HTTP Server或IIS，作为独立服务器运行。
    Node.js下载地址。安装node.js会包含环境变量以及npm的安装，安装后，检测Node.js是否安装成功输入：
    node -v
    检测npm是否安装成功，输入：
    npm -v
    至此，完成了hexo框架所依赖的Node.js环境，下一步就可以安装hexo框架了。

  • hexo：读作“嗨克搜”，它就是基于Node.js开发的个人博客网站搭建的框架程序，我们需要在电脑里创建一个日后用于存储博客的目录，命名为blog，以后hexo框架和自己编写的博客都位于该目录及其子目录下。创建好后，进入blog目录，仍旧是右键打开git bash。
    使用npm包管理器安装hexo
    npm install -g hexo-cli
    初始化博客
    hexo init
    测试博客网站雏形：
    hexo new test_blog // 默认会有个hello world，可以不创建博客
    hexo g        // 生成静态页面
    hexo s        // 启动本地服务器
    浏览器输入地址localhost:4000查看博客网页

问题记录

markdown博客渲染失败，只显示空白页问题

因为当时卸载了默认的markdown解析器，安装了一个新的解析器，可能是新的解析器配置问题，导致markdown博客无法解析，只显示空白页，所以我后来重装原有的解析器问题解决了。
npm i hexo-renderer-marked --save

hexo 本地调试

hexo s --debug 该命令可以帮助在发布之前调试，debug模式会输出一些可能的错误信息日志，帮助定位问题所在。

解决方案链接
hexo-toc 这个插件导致的，这应该是其它主题需要的依赖吧，NexT 不需要，所以删除它！
